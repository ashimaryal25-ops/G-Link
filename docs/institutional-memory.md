# Institutional Memory

Student clubs have a structural turnover problem. Presidents, secretaries, treasurers, and event chairs change often, but the club still has recurring events, traditions, funding rules, vendor relationships, planning timelines, and avoidable mistakes.

G-Link should preserve the club's operating memory so new officers can learn how the club used to run and make better decisions faster.

## Core Principle

The archive is not a dead file cabinet. It should be searchable, connected to real records, and useful when planning the next event.

## What Should Be Archived

- Approved meeting minutes.
- Decisions and motions.
- Completed and cancelled events.
- Event timelines.
- Task lists and task comments.
- Assigned owners.
- Budgets, actual spending, receipts, and reimbursements.
- Communication drafts, plus sent-message records once email integrations exist.
- Collaboration history with other clubs.
- Proposal reviews and audit findings.
- Final event reports.
- Officer handoff notes.

## Archive Views

### Club Timeline

Shows the club's history by term or academic year:

- Officers for that term.
- Meetings held.
- Events planned.
- Total budget requested and spent.
- Major decisions.
- Reports generated.

### Event Archive

Shows completed events as reusable examples:

- What the event was.
- When planning started.
- Which tasks were needed.
- Which members owned the work.
- What the budget looked like.
- Which vendors or partners were used.
- What went wrong.
- What should be repeated.

### Role Handoff

Shows incoming officers what their role needs:

- Previous officer notes.
- Recurring responsibilities.
- Open tasks.
- Important deadlines.
- Common mistakes.
- Relevant old events and reports.

## Recurring Event Playbooks

Many clubs repeat events every semester or year. G-Link should let officers clone a completed event into a new event.

When cloning, the app should offer:

- Prior task list.
- Prior budget lines.
- Prior vendors.
- Prior communication drafts.
- Prior collaboration partners.
- Prior audit issues.
- Prior lessons learned.

The new event should be a fresh record, but it should link back to the archived event it came from.

## AI Use

AI can make the archive more useful by summarizing:

- How this club usually runs meetings.
- How long recurring events usually take to plan.
- What budget ranges were normal.
- Which tasks were commonly forgotten.
- What advice prior officers left for the next board.

AI should not overwrite original records. It should generate summaries and playbooks that link back to the source minutes, events, tasks, budgets, and comments.
